# Mode 4 Spec: ARMORY

Working title: `ARMORY`

`Classic` がライン消し、`Bomber` が爆弾連鎖、`Purify` がクラスター浄化なら、`ARMORY` は `絵完成 -> 攻撃発動` のモードにする。

## Core Pitch

- 落ちてくるのは色ブロックではなく、武器や攻撃道具の「絵の断片」。
- 盤面上でひとつの絵が完成すると、その絵に対応した攻撃エフェクトが発動する。
- 攻撃は盤面上の不要セルを破壊し、さらに別の絵が完成すると連鎖する。
- 落下ピースには別アイテムの断片が混ざるので、ただ積むだけでは狙った絵が完成しない。

## Design Goals

- 今の3モードとルールの芯が被らないこと。
- 見た瞬間に「何を完成させたか」「どんな攻撃が出るか」が分かること。
- モバイルでも視認できること。
- 実装がデータ駆動で増やしやすいこと。

## Player Fantasy

- ただ消すのではなく「組み立てて撃つ」感覚を出す。
- 盤面が武器工場や即席兵器アセンブリのように見える。
- 完成した絵のシルエットに沿って攻撃が走ることで、演出に意味を持たせる。

## Core Rules

1. 落下ピースは 2-3 セルの断片セット。
2. 各セルは `weaponId` と `fragmentId` を持つ。
3. 盤面上で同じ `weaponId` の必要断片が所定パターンでそろうと、その武器が完成。
4. 完成した武器は短いチャージのあと、武器ごとの攻撃エフェクトを発動。
5. エフェクト範囲内のセルを破壊。
6. 重力で落下。
7. その結果また別の武器が完成したら連鎖。

## Recommended MVP Scope

- 武器はまず 4 種類だけ。
- レシピサイズは `3x3` 固定で始める。
- まずは `正位置のみ成立` にする。
- まずは `盤面セルの破壊` のみにする。別レイヤーの敵や弾は後回し。
- 1 ピースあたり 3 セル以下。

これで十分に新規性が出るし、実装負荷もまだ管理できる。

## Weapon Set

### 1. Missile

- Role: 単線の貫通攻撃
- Visual: ミサイル本体のシルエットに沿って点火、煙の尾を引いて発射
- Effect:
  - 完成位置から上方向へ一直線に貫通
  - 同じ列、または完成中心列を細く抜く
- Use:
  - 縦穴や高い塔を崩す
  - ピンポイント除去役

### 2. Bomb

- Role: 円形の範囲破壊
- Visual: 導火線が短く燃えてから爆発、リングと火花を出す
- Effect:
  - 完成中心から半径 2 前後を破壊
  - 最も分かりやすい万能武器
- Use:
  - 固まり除去
  - 連鎖の始点

### 3. Tub

- Role: 重量プレス
- Visual: たらいが上から落ちてきて、着地点を押し潰す
- Effect:
  - `3x2` か `3x3` の横広い圧殺
  - 着地点中心に短い衝撃波
- Use:
  - 凹凸をまとめて壊す
  - 広めの平地作り

### 4. Frying Pan

- Role: 横なぎ払い
- Visual: フライパンの軌道に沿って弧を描くスイング
- Effect:
  - 完成位置を中心に横方向の扇形か帯状を破壊
  - 斜め前方にも少し届くと気持ちいい
- Use:
  - 横に広い詰まりの処理
  - 連鎖のつなぎ

## Visual Rules

- 断片絵は細かいイラストではなく、`太い輪郭 + 1つの特徴` にする。
- 1 武器あたり色は 2 色まで。
- 断片の境界は見えるように外枠を入れる。
- `あと 1 ピース` の完成形には薄いゴースト輪郭を出すと遊びやすい。
- 完成時はまずシルエットが発光し、その輪郭に沿って攻撃を走らせる。

## Why 3x3 Is Recommended

- スマホでも読める。
- 絵合わせの達成感がギリギリ残る。
- レシピ判定が軽い。
- UI や NEXT 表示にも載せやすい。

`4x3` 以上は Phase 2 以降でいい。

## Piece Composition Rules

- 1 ピースは `同一武器だけ` ではなく、複数武器の断片が混ざる。
- これにより「欲しい絵を作るための置き順」が生まれる。
- ただし完全ランダムだと読めないので、以下を推奨:
  - 1 ピース内で最大 2 武器まで
  - 6 手以内には必ず現在進めている武器の断片が 1 回は来る
  - 盤面が苦しい時は、Bomb か Tub の断片を少し優遇する

## Match Detection

各セルが単なる色ではなく `weaponId + fragmentId` を持つ前提で、ロック後に以下を行う。

1. 全レシピの原点候補を走査
2. 必要マスに必要 fragment がそろっているか確認
3. 成立したら `ResolvedWeapon` としてキューへ積む
4. 同フレームで複数成立したら順番に発火
5. 破壊 -> 重力 -> 再走査

## Chain Design

- 同時完成 2 個以上で `DOUBLE ARMORY` 的なボーナス演出
- 連鎖数に応じて倍率上昇
- 同種連打より、異種連鎖のほうが演出的に派手にしやすい

例:

- Missile -> Bomb: 列を抜いた結果、爆弾が完成して範囲爆破
- Frying Pan -> Tub: 横なぎ払いで断片がそろい、重しで押し潰す

## HUD / UX

- 現在作りかけの武器を 1-2 個だけサイドに表示
- `あと何断片で完成か` を見せる
- 完成直前の武器には盤面上で淡い枠を出す
- 初期版ではアイテム数を欲張らない

## Scoring Proposal

- 武器完成ボーナス: 小
- 破壊セル数ボーナス: 中
- 連鎖倍率: 大
- 複数武器同時発動ボーナス: 大

つまり「単発完成」より「攻撃で次の武器がそろう」ほうが強い設計にする。

## Technical Shape

Recommended runtime data:

```ts
type WeaponId = 'missile' | 'bomb' | 'tub' | 'pan';

type FragmentCell = {
  weaponId: WeaponId;
  fragmentId: string;
};

type WeaponRecipe = {
  id: WeaponId;
  width: number;
  height: number;
  cells: (string | null)[][];
  effect: 'line' | 'blast' | 'slam' | 'sweep';
};
```

High-level engine flow:

1. ピースをロック
2. レシピ成立チェック
3. 成立武器を演出キューへ
4. 攻撃範囲のセル破壊
5. 重力
6. 再スキャン

## Rendering Notes

- エフェクトは単なる四角パーティクルではなく、完成した絵のシルエットに沿わせる。
- Missile:
  - ノズル点火 -> 煙 -> 発射線
- Bomb:
  - 導火線 -> 閃光 -> 爆風リング
- Tub:
  - 上空影 -> 落下 -> ドン -> 横波
- Pan:
  - 風切り弧 -> 打撃フラッシュ -> 火花

## What To Avoid

- 写実的すぎる絵
- 1 モード目から武器を 8 種類以上入れる
- 小さな差分で見分けるデザイン
- 完成条件が複雑すぎて「何を狙うゲームか」分からなくなること

## Phase Plan

### Phase 1

- 4 武器
- 3x3 固定
- 正位置のみ
- 盤面セル破壊のみ
- 連鎖あり

### Phase 2

- レシピ回転対応
- 武器ごとの向き対応エフェクト
- 特殊敵や硬い障害物
- 武器どうしの合成ボーナス

## Recommendation

まずは `Bomb` 1 種だけでプロトタイプを作るのが一番安全。

理由:

- 完成判定が分かりやすい
- エフェクトが作りやすい
- 盤面破壊で連鎖が出るかを見やすい
- このモードの気持ちよさの核がすぐ分かる

その後に `Missile`、`Tub`、`Frying Pan` を足していく。
